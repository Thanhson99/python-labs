# Services

This folder groups each lab project as an independent service-like module.

## Naming Convention

- Keep each service isolated in its own folder.
- Prefer one clear entrypoint (`main.py`, `app.py`, or `index.py`).
- Keep service-specific dependencies in local `requirements.txt` when needed.
